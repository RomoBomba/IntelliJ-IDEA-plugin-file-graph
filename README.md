# File Graph – визуализация зависимостей файлов проекта в IntelliJ IDEA

Интерактивный плагин, который выводит граф файлов (Java / Kotlin) вашего проекта, показывает количество связей, указывает на циклические зависимости и позволяет быстро открывать нужный файл прямо из графа.

---

## Основные возможности

| Функция | Описание |
|---------|----------|
| **Построение графа** | Сканирует проект, находит все `.java`, `.kt`, `.kts` файлы и строит связи на основе `import`, `extends`, `implements` |
| **Поиск** | Фильтрация узлов по имени/пути с помощью строки поиска |
| **Фильтры** | Показ/скрытие `.java`, `.kt` файлов, циклических зависимостей, изолированных файлов |
| **Циклические зависимости** | Узлы и рёбра, входящие в цикл, выделяются красным цветом |
| **Зум** | Колёсико мыши, трекпад, кнопки + / – / ⟲ в header |
| **Автообновление** | При изменении файлов граф обновляется автоматически (debounce ~1.5с) |
| **Клик для открытия** | Одинарный клик выделяет файл, двойной — открывает в редакторе |

## Как работает

### Архитектура

```
┌─────────────────────────────────────────────────────────────┐
│                     IntelliJ IDEA                           │
│  ┌─────────────────┐  ┌──────────────┐  ┌───────────────┐  │
│  │ GraphToolWindow │──│  GraphPanel  │──│  JcefBridge   │  │
│  │    Factory      │  │              │  │               │  │
│  └─────────────────┘  └──────┬───────┘  └───────┬───────┘  │
│                              │                  │          │
│                     ┌────────▼────────┐         │          │
│                     │DependencyGraph  │         │          │
│                     │   Service       │         │          │
│                     │  (PSI analysis) │         │          │
│                     └────────┬────────┘         │          │
│                              │                  │          │
│                     ┌────────▼────────┐  ┌──────▼──────┐  │
│                     │  ForceGraph.js  │──│   index.html│  │
│                     │   (webview)     │  │  (UI + D3)  │  │
│                     └─────────────────┘  └─────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### Поток данных

1. **PSI Scanning** — `DependencyGraphService` использует IntelliJ PSI для анализа Java/Kotlin файлов:
   - `FileTypeIndex.getFiles()` — находит все исходные файлы проекта
   - `PsiJavaFile.importList` — извлекает импорты
   - `PsiClass.extendsList/implementsList` — извлекает наследование
   - `KtFile.importDirectives` — извлекает импорты из Kotlin

2. **Graph Construction** — строится направленный граф:
   - Узлы = файлы проекта
   - Рёбра = зависимости (import, extends, implements)
   - Циклы обнаруживаются алгоритмом DFS

3. **WebView Rendering** — данные передаются в браузер через JCEF:
   - `ForceGraph` (D3.js force simulation) отображает граф
   - Canvas API используется для кастомной отрисовки узлов
   - Ребра показывают направление стрелками

## Ограничения

| Параметр | Значение | Описание |
|----------|----------|----------|
| **Timeout сканирования** | 60 сек | Если сканирование превышает 60 секунд, оно автоматически отменяется |
| **Кластеризация** | > 800 узлов | При превышении 800 узлов мало связанные файлы группируются в один узел |
| **Debounce** | 1.5 сек | Задержка перед автообновлением при изменениях файлов |
| **Dependencies/file** | ≤ 100 | Лимит зависимостей, извлекаемых из одного файла |

### Производительность

- **До 500 файлов** — мгновенное отображение (<100мс)
- **500-1000 файлов** — 1-3 секунды
- **1000-3000 файлов** — 5-15 секунд (зависит от связности проекта)
- **> 3000 файлов** — рекомендуется использовать фильтры

## Установка

### Через Marketplace (пока недоступно)

1. Откройте **Settings → Plugins**
2. Выберите **Marketplace**, введите `File Graph`
3. Установите и перезапустите IDE

### Через артефакт

1. Скачайте `file-graph-<version>.zip` с GitHub Releases
2. **Plugins → ⚙️ → Install Plugin from Disk…**
3. Перезапустите IDE

---

## Использование

1. **View → Tool Windows → File Graph**
2. Дождитесь статуса "Connected"
3. **Поиск** — введите имя файла в строку поиска
4. **Фильтры** — используйте чекбоксы для фильтрации:
   - `.java` — показать/скрыть Java файлы
   - `.kt` — показать/скрыть Kotlin файлы
   - `Circular` — показать/скрыть только циклические зависимости
   - `Isolated` — показать файлы без связей
5. **Клик** — выделение и подсветка соседей
6. **Двойной клик** — открыть файл в редакторе
7. **Refresh** в toolbar — принудительное пересканирование

## Сборка

```bash
# Клонировать и перейти в директорию
cd file-graph

# Синхронизировать зависимости
./gradlew clean

# Запустить в режиме разработки
./gradlew runIde

# Или собрать плагин
./gradlew buildPlugin
# Артефакт: build/distributions/file-graph-*.zip
```

**Требования:** JDK 21+, Gradle 8+, IntelliJ IDEA 2026.1+
